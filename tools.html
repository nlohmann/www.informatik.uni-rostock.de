BPEL2oWFN

  • BPEL2oWFN translates WS-BPEL processes into a Petri net model. It implements a feature-complete Petri net semantics for the first specification proposal BPEL4WS 1.1 and the current specification WS-BPEL 2.0.
  • Niels Lohmann. A Feature-Complete Petri Net Semantics for WS-BPEL 2.0. In Marlon Dumas and Reiko Heckel, editors, Web Services and Formal Methods, Forth International Workshop, WS-FM 2007, Brisbane, Australia, September 28-29, 2007, Proceedings, volume 4937 of Lecture Notes in Computer Science, pages 77–91, April 2008. Springer-Verlag. DOI
  • co-developers: Christian Gierds, Dennis Reinert, and Martin Znamirowski
  • development status: inactive
  • latest release: June 29, 2007

Petri Net API

  • The Petri Net API encapsulates Petri net-related functions in a C++ library. It is currently used in many of the service-technology.org tools such as Mia and Wendy. The functionality includes creation, deletion, and manipulation of Petri net nodes, import and export using many standard Petri net file formats including PNML, application of structural reduction rules, and several structural checks, for instance for the free-choice property or workflow net structure.
  • Niels Lohmann, Stephan Mennicke, and Christian Sura. The Petri Net API: A collection of Petri net-related functions. In Martin Schwarick and Monika Heiner, editors, Proceedings of the 17th German Workshop on Algorithms and Tools for Petri Nets (AWPN 2010), Cottbus, Germany, October 7-8, 2010, volume 643 of CEUR Workshop Proceedings, pages 148–155, October 2010. CEUR-WS.org.
  • co-developers: Stephan Mennicke, Georg Straube, Christian Sura, and Robert Waltemath
  • development status: active
  • latest release: July 28, 2010
